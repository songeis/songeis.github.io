# 对链表的误解

在没有尝试实现链表的时候，我一直以为链表从结构上将应该是一个对象。因为链表不就是一个有序的元素集合嘛。再加上所有关于链表的图都是像下面这样画的。

所以我就误以为链表实际上是可以靠一个对象实现的。

{
	head:{element:xx,next:xx},
	"1":{element:xx,next:xx},
	"2":{element:xx,next:xx},
	...
	"n":{element:xx,next:null},
}

但其实仔细想想这样的话，数据完全没有串起来(或者说"链起来"),而且这样的方式数据会很容易被修改。

我们对照着节点结构图看一下，节点都有两部分内容组成：value 和 next。虽然头节点和尾节点有点特殊，但到底还是节点所以也应该有节点的结构。

# 空链表是什么状态

我们刚才说了节点由两部分组成：value+next

那么链表又由什么组成呢？

链表也由两部分组成；节点+链。

这里特别说一下链，链就是用来将节点从头到尾串起来。

在这里，我们可以将链表想成一个火车。空链表相当于没有车厢，只有车头和车尾的火车，既然没有乘客，那么车尾也可以省略，只有车头调头会更容易些。

但车头也是一节车厢（即一个节点）也有车厢的结构（节点结构）。而且不管有没有乘客，这个车头都一定是存在的。

所以按照这种方式，我们可以先实例化一个车头。

class Train {
  constructor() {
    this.head = {
      value: undefined,
      next: null
    }
  }
}
let hexie = new Train()
console.log(hexie)

为了更清楚的看到链表的结构，我们再思考一下含有一个节点的链表是什么样的。

head指针会指向这唯一的一个节点。而这个节点的 next指向 尾节点（新增）

好，现在我们就把头等舱和二等舱安上。

class Train {
  constructor() {
    this.head = {
      value: undefined,
      next: null
    }
    this.firstClassCarriages = {
      value: "leo",
      next: null
    }
    this.secondClassCarriages = {
      value: "cat",
      next: null
    }
  }
  addNode() {
    this.firstClassCarriages.next = this.secondClassCarriages
    this.head = this.firstClassCarriages
    return this.head
  }
}
let hexie = new Train()
console.log(hexie)
console.log(hexie.addNode())

这个时候我们看到实际上我们说的链表数据结构是长成这个样子的

{ value: 'leo', next: { value: 'cat', next: null } }
